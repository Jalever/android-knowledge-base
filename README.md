# Android 团队技术知识库

## 项目简介

本知识库是 Android 团队的内部技术沉淀 wiki，系统性地整理项目中涉及的核心技术领域，帮助团队建立统一的技术认知、沉淀实践经验、加速问题排查。

## 知识模块

| 编号 | 模块 | 说明 |
|------|------|------|
| 01 | 国际化 | 多语言翻译、布局适配、后端环境切换、TTS 多语言 |
| 02 | 视频播放 | 播放器选型、ExoPlayer/Media3、视频性能优化 |
| 03 | 蓝牙通信 | BLE 低功耗蓝牙、经典蓝牙主从模式 |
| 04 | 串口通信 | 串口协议基础、Android 串口实现方案 |
| 05 | 系统稳定性 | 死机重启恢复、稳定性调试方法论 |
| 06 | WiFi 稳定性 | WiFi 连接管理、断连排查与重连策略 |
| 07 | 性能优化 | 内存优化、UI 渲染性能、启动优化 |

## 知识整理规范

本知识库遵循 **三层结构** 组织，所有团队成员在新增或修改文档时请遵守以下规范。

### 三层结构

#### 第一层：目录（INDEX）

全局导航入口，位于根目录 [INDEX.md](INDEX.md)。

- 按模块分类，列出所有知识模块及其子主题
- 标注每篇文档的完成状态（已完成 / 进行中 / 待补充）
- 提供快速跳转链接

#### 第二层：概要（Overview）

每个模块目录下的 `00-overview.md`，定位为该技术领域的"一页纸"综述。

概要应覆盖：

- **核心原理**：这项技术是什么、为什么需要它
- **发展趋势与版本演进**：Android 各版本间的 API 变化与迁移路径
- **主流方案与开源项目对比**：当前可选方案、各方案优劣与社区活跃度
- **适用场景与选型建议**：什么情况下用哪种方案
- **快速上手路径**：推荐的学习顺序和模块内文档阅读优先级

篇幅控制：概要应控制在速览 1-2 页即可了解全貌。

#### 第三层：详情（Detail）

模块目录下 `01-*.md`、`02-*.md` 等编号文件，每个文件聚焦一个具体子主题。

详情应覆盖：

- **内部流程**：配合流程图 / Mermaid 图解，说明技术实现的内部运作机制
- **关键 API 与代码片段**：核心 API 用法，以 Kotlin 为主，必要时附 Java
- **常见坑点与排查思路**：实际开发中遇到的问题及解决方案
- **踩坑记录**：团队成员在项目中遇到的真实案例（持续补充）
- **参考资料与延伸阅读**：官方文档、优质博客、相关开源项目链接

### 文档元信息

每篇文档（概要和详情）头部必须包含以下元信息区：

```markdown
---
创建日期: YYYY-MM-DD
最后更新: YYYY-MM-DD
作者: <姓名>
状态: [草稿 | 进行中 | 已完成]
前置知识: <阅读本文前建议先了解的内容>
关联模块: <与本文相关的其他模块>
---
```

### 文件命名规范

- 模块目录：`{两位序号}-{英文短横线命名}`，如 `03-bluetooth/`
- 概要文件：`00-overview.md`
- 详情文件：`{两位序号}-{英文短横线命名}.md`，如 `01-ble-communication.md`
- 序号体现推荐阅读顺序

### 写作原则

- 概要求"广"：快速建立全局认知，不深入细节
- 详情求"深"：以"能指导实际开发"为标准，不追求面面俱到
- 代码片段以 Kotlin 为主，必要时附 Java 版本
- 善用 Mermaid 图表辅助说明流程和架构
- 时间有限时，详情可先列提纲 + 关键代码，标记状态为"草稿"，后续迭代补充
- 鼓励团队成员在"踩坑记录"区域持续补充项目中的真实案例
